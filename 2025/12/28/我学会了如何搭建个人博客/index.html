我学会了如何搭建个人博客

发表于2025-12-28|更新于2025-12-29|学习笔记
|浏览量:

就在刚刚，我成功搭建了属于自己的个人博客，使用 Hexo + Butterfly 主题。下面是整个过程以及我遇到的一些阻碍和解决方法。

  1. 准备工作
  • 我选择在本地 D 盘创建博客文件夹：
    D:\BlogProjects

mathematica

  • 注意从 C 盘命令行切换到 D 盘时，需要执行：
    D:
    cd D:\BlogProjects

markdown

  • 安装 Hexo 前确保已安装 Node.js 和 npm。
  1. 初始化 Hexo
  • 在空目录下执行：
    hexo init .
    npm install
    Hexo 会自动生成基础目录结构，包括 _config.yml、source、themes 等。

如果目录不为空，hexo init 会报错，需要先清空文件夹或者备份重要内容。

  1. 安装 Butterfly 主题
    进入 themes 文件夹：

cd themes
克隆 Butterfly 主题：

git clone https://github.com/jerryc127/hexo-theme-butterfly.git butterfly
返回博客根目录，修改 _config.yml：

yaml
复制代码
theme: butterfly
安装主题依赖：

复制代码
npm install hexo-butterfly –save
npm install hexo-renderer-pug hexo-renderer-sass –save

  1. 新建第一篇文章
    使用命令：

hexo new “我学会了如何搭建个人博客”
Hexo 会在：
D:\BlogProjects\source_posts
生成 Markdown 文件：我学会了如何搭建个人博客.md

  1. 编辑文章内容
    打开 Markdown 文件，写下博客搭建心得：

安装 Hexo

安装 Butterfly 主题

本地预览博客

写文章流程

  1. 本地生成与预览
    执行：

hexo clean
hexo generate
hexo server

打开浏览器访问：
http://localhost:4000/
可以在首页看到刚创建的文章。

通过以上步骤，我成功搭建了自己的个人博客，并学会了使用 Hexo + Butterfly 主题展示内容。
在操作过程中遇到了一些插件版本和跨盘符的阻碍，但都可以通过命令行和配置调整顺利解决。

搭建个人博客遇到的挫折整理

  1. 安装 Hexo 主题和插件报错

    在本地执行 hexo new 创建文章时，控制台提示：

    说明 Butterfly 主题插件加载失败，初期不知道原因，导致 Hexo 在部分命令执行时总是报错。

  2. 跨盘符操作不熟悉

    初次在 C 盘安装 Hexo，但项目目录在 D 盘。需要手动切换盘符（例如输入 D:），否则命令无法找到项目路径。
    对 Hexo 命令行工作目录和系统路径不熟悉，导致执行命令时容易出错。

  3. 版本兼容问题

    安装 Node.js、Hexo、Hexo 插件时，不同版本存在兼容性问题，有些插件无法在当前 Node 版本下正常使用，报错信息不够直观，需要花时间排查。

  4. 文章默认显示“无标题”

    创建第一篇文章时，即使在 hexo new "文章标题" 中指定了标题，网页仍然显示“无标题”，原因是 Markdown 文件的 front-matter 格式不正确，或者主题配置中未识别标题字段。

  5. 默认用户名和头像需要修改

    Hexo 默认用户名是 John Doe，头像也是默认图片，需要修改 _config.yml 才能显示自己的昵称和头像。

  6. 博客在本地和 GitHub Pages 上效果不一致

    本地 hexo s 启动时，样式和内容都正常，但部署到 GitHub Pages 后，CSS 和 JS 丢失，所有超链接点击后显示 404。
    原因是没有配置 root，Hexo 默认以 / 为根路径，而项目仓库页面 URL 是 https://username.github.io/仓库名/，导致路径不匹配。

  7. GitHub Pages 部署初期配置麻烦

    部署到 GitHub Pages 时，需要安装 hexo-deployer-git 并设置仓库地址和分支。
    第一次部署报错：

    需要手动运行 git config --global user.name "Your Name" 和 git config --global user.email "you@example.com" 来设置 Git 账号信息。

  8. 第一次在 GitHub 上的页面缺乏美化

    即使部署成功，访问 GitHub Pages 页面时没有 CSS 样式，博客看起来像纯文本。
    需要修改 _config.yml 添加 root 字段，清理缓存并重新生成、部署才能正常显示。

  9. 许可证选择困惑

    在创建 GitHub 仓库时，对于是否添加 License 以及选择哪种许可证不清楚。
    虽然可以先选择 MIT 或者其他随意许可证，但仍需要理解 Creative Commons 或者 MIT 的区别。

  10. Hexo 命令报 LF/CRLF 警告

    部署过程中控制台提示：

    虽然不影响功能，但显示很多警告信息，需要了解 Git 的换行符处理机制。

心得体会

经过这次搭建个人博客的过程，我深刻体会到技术实践中不可避免的挫折与挑战。从最初的环境配置、插件安装，到主题适配、路径调整，每一个环节都可能遇到意料之外的问题。尤其是 Hexo 与 Butterfly 主题的兼容问题，以及本地路径与 GitHub Pages 部署路径的差异，让我不得不多次查阅文档和调试配置文件。这不仅锻炼了我的问题排查能力，也让我对博客系统的运行机制有了更清晰的理解。

同时，我学会了如何使用 Git 管理博客项目，从设置本地用户名、邮箱，到推送到 GitHub，再到配置 GitHub Pages，实现了本地写作与远程展示的完整流程。通过反复生成、部署和调试，我掌握了 Hexo 的基本命令与配置技巧，也理解了网页路径、资源引用和前端样式加载之间的关系。

最重要的是，这次实践让我认识到，任何技术项目的完成都离不开耐心和持续尝试。遇到报错或不符合预期的情况时，冷静分析、查阅资料和尝试不同方案，往往是解决问题的关键。虽然过程曲折，但每一次调试成功都带来了成就感，也让我更加自信地面对未来的技术挑战。通过这次博客搭建，我不仅完成了一个可以长期更新和展示的个人网站，更积累了宝贵的实战经验，为后续的学习和创作打下了坚实基础。
