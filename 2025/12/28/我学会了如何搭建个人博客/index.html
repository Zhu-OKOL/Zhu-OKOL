我学会了如何搭建个人博客

发表于2025-12-28|更新于2025-12-28|学习笔记
|浏览量:

就在刚刚，我成功搭建了属于自己的个人博客，使用 Hexo + Butterfly 主题。下面是整个过程以及我遇到的一些阻碍和解决方法。

  1. 准备工作
  • 我选择在本地 D 盘创建博客文件夹：
    D:\BlogProjects

mathematica

  • 注意从 C 盘命令行切换到 D 盘时，需要执行：
    D:
    cd D:\BlogProjects

markdown

  • 安装 Hexo 前确保已安装 Node.js 和 npm。
  1. 初始化 Hexo
  • 在空目录下执行：
    hexo init .
    npm install
    Hexo 会自动生成基础目录结构，包括 _config.yml、source、themes 等。

如果目录不为空，hexo init 会报错，需要先清空文件夹或者备份重要内容。

  1. 安装 Butterfly 主题
    进入 themes 文件夹：

cd themes
克隆 Butterfly 主题：

git clone https://github.com/jerryc127/hexo-theme-butterfly.git butterfly
返回博客根目录，修改 _config.yml：

yaml
复制代码
theme: butterfly
安装主题依赖：

复制代码
npm install hexo-butterfly –save
npm install hexo-renderer-pug hexo-renderer-sass –save

遇到的阻碍
1.EISDIR 错误

每次运行 hexo clean 或 hexo server 时，都会提示：
ERROR Plugin load failed: hexo-butterfly
Error: EISDIR: illegal operation on a directory, read
经过测试，这个错误不会影响博客生成或本地预览，所以可以暂时忽略。

2.插件版本不兼容

一些 npm 包会有警告或弃用提示，例如 core-js、cuid 等。

使用 npm install 后依赖仍可正常运行，只是提示存在潜在安全或兼容问题。

3.跨盘符命令行问题

初次在 C 盘命令行操作 Hexo 时，直接 cd D:\BlogProjects 并不会切换盘符，需要先输入 D:，然后再 cd。

  1. 新建第一篇文章
    使用命令：

hexo new “我学会了如何搭建个人博客”
Hexo 会在：
D:\BlogProjects\source_posts
生成 Markdown 文件：我学会了如何搭建个人博客.md

  1. 编辑文章内容
    打开 Markdown 文件，写下博客搭建心得：

安装 Hexo

安装 Butterfly 主题

本地预览博客

写文章流程

  1. 本地生成与预览
    执行：

hexo clean
hexo generate
hexo server

打开浏览器访问：
http://localhost:4000/
可以在首页看到刚创建的文章。

通过以上步骤，我成功搭建了自己的个人博客，并学会了使用 Hexo + Butterfly 主题展示内容。
在操作过程中遇到了一些插件版本和跨盘符的阻碍，但都可以通过命令行和配置调整顺利解决。

搭建个人博客不仅让我掌握了 Hexo 的基本用法，也让我理解了主题安装、依赖管理和本地预览的重要流程。
